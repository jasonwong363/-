# 蓝 V 体系 SOP｜表格化执行版

> 蓝 V 体系是公司 SOP 的第一块样板。  
> 目标是把“蓝 V 怎么做”拆成：客户资料、对标、定位、关键词、百问百答、内容分类、爆文改帖、投放、数据复盘等可执行表格。

---

## 01. 蓝 V 体系总流程

```text
客户资料收集
→ 找对标
→ 对标表拆解
→ 百问百答
→ 客户红线表
→ 定位表
→ 关键词表
→ 内容类型分配
→ 内容公式拆解
→ 爆文改帖
→ 投放计划
→ 发布排期
→ 数据复盘
→ 智能体更新
```

---

## 02. 蓝 V 核心认知

### 老麦判断

```text
蓝 V 很简单：一个找对标，一个抄对标，核心是把表格做出来。
```

### 正确理解

这不是低级抄袭，而是：

| 动作 | 正确含义 |
|---|---|
| 找对标 | 找出平台已验证过的内容、人群、话题、账号和转化路径 |
| 抄对标 | 拆解其结构、关键词、标题、内容比例、评论区和转化方式 |
| 做表格 | 把对标拆成标准字段，让员工和 AI 可复制执行 |

# 第一模块：客户资料收集

## 03. 客户基础资料

客户进入蓝 V 体系前，必须提供基础资料。

| 资料 | 用途 |
|---|---|
| 营业执照 | 蓝 V 认证 |
| 身份证 / 法人资料 | 平台认证资料 |
| 品牌介绍 | 了解客户是谁 |
| 产品资料 | 明确卖什么 |
| 服务流程 | 明确交付什么 |
| 价格体系 | 方便内容和销售话术 |
| 案例资料 | 用于案例型内容 |
| 客户红线 | 避免写错或违规 |
| 过往内容 | 判断账号基础 |
| 竞品名单 | 辅助对标 |

---

## 04. 客户资料交付规则

```text
客户资料没交齐，不进入正式内容生产。
客户红线没确认，不进入批量发布。
百问百答没完成，不进入智能体投喂。
提案没确认，不进入正式执行周期。
```

# 第二模块：找对标

## 05. 找对标的四个定位目的

根据会议内容，找对标的目的包括：

```text
人群定位
画像定位
产品定位
话题定位
```

继续延展为：

| 定位 | 要回答的问题 |
|---|---|
| 人群定位 | 谁会买？谁最需要？谁最有付费能力？ |
| 画像定位 | 他们年龄、城市、职业、预算、痛点是什么？ |
| 产品定位 | 产品解决什么问题？和竞品差异在哪？ |
| 话题定位 | 什么话题能吸引这类人？ |
| 关键词定位 | 他们会搜索什么词？ |
| 内容定位 | 他们更愿意看知识、案例、避坑还是产品？ |

---

## 06. 对标账号筛选标准

| 维度 | 选择标准 |
|---|---|
| 行业相似度 | 是否同品类 / 同城市 / 同人群 |
| 内容数据 | 是否有高互动笔记 |
| 商业意图 | 是否在引流 / 转化 / 投放 |
| 内容类型 | 是否覆盖知识、案例、产品、营销等多类型 |
| 评论区质量 | 是否有真实客户问题 |
| 账号稳定性 | 是否长期更新 |
| 可复制度 | 是否能拆成模板 |

---

## 07. 对标拆解字段

每个对标账号至少拆：

| 字段 | 说明 |
|---|---|
| 对标账号名 | 账号名称 |
| 平台链接 | 主页链接 |
| 账号定位 | 它对外讲自己是谁 |
| 目标人群 | 它写给谁看 |
| 主要产品 | 它卖什么 |
| 核心卖点 | 它反复强调什么 |
| 内容类型占比 | 知识/案例/产品/营销等比例 |
| 爆款笔记标题 | 高数据内容标题 |
| 封面特点 | 首图视觉和标题风格 |
| 正文结构 | 开头、正文、结尾、CTA |
| 关键词 | 标题、正文、标签高频词 |
| 评论区问题 | 用户最常问什么 |
| 转化方式 | 私信、主页、表单、评论引导 |
| 可借鉴点 | 能复制什么 |
| 避免点 | 不能照搬什么 |

# 第三模块：百问百答

## 08. 百问百答的定义

百问百答不是简单 FAQ，而是客户产品、服务、优势、缺点、红线和成交理由的完整资料库。

### 会议中的定义

```text
客户问你是卖什么产品的？
客户问你为什么要找你买单的 100 个理由。
产品的百问百答，服务的百问百答。
```

---

## 09. 百问百答的位置

```text
找完对标之后
→ 帮客户做案例之前
→ 做智能体之前
→ 做内容批量生产之前
```

---

## 10. 百问百答的五类问题

| 类型 | 说明 |
|---|---|
| 产品问题 | 产品是什么、怎么用、多少钱、和别人有什么不同 |
| 服务问题 | 服务流程、周期、交付、售后、保障 |
| 信任问题 | 为什么选你、有什么案例、凭什么相信 |
| 决策问题 | 适不适合我、贵不贵、有没有必要 |
| 风险问题 | 有什么缺点、不能承诺什么、客户会担心什么 |

# 第四模块：内容体系

## 11. 蓝 V 内容类型

| 类型 | 核心作用 | 输出方向 |
|---|---|---|
| 知识型 | 建立专业信任 | 原理、流程、判断标准 |
| 科普型 | 降低认知门槛 | 小白入门、常识纠偏 |
| 花絮型 | 提升真实感 | 公司现场、交付过程、团队工作 |
| 故事型 | 建立代入感 | 客户经历、老板故事、项目过程 |
| 产品型 | 讲清产品价值 | 卖点、场景、对比、细节 |
| 案例型 | 证明能力 | 客户前后变化、项目复盘 |
| 营销型 | 推动转化 | 活动、优惠、咨询引导 |
| 对比避坑型 | 借痛点教育用户 | 红黑榜、避坑、误区、选择标准 |

---

## 12. 内容二级拆分示例

### 知识型

| 二级类型 | 适合内容 |
|---|---|
| 行业认知型 | 这个行业到底怎么回事 |
| 流程解释型 | 一件事从开始到结束怎么做 |
| 原理拆解型 | 为什么会这样 |
| 误区纠偏型 | 很多人理解错在哪里 |
| 选择标准型 | 怎么判断好坏 |
| 术语解释型 | 专业词翻译成人话 |

### 科普型

| 二级类型 | 适合内容 |
|---|---|
| 小白入门型 | 适合完全不了解的人 |
| 常识型 | 用户应该知道的基础信息 |
| 价格科普型 | 为什么贵 / 为什么便宜 |
| 流程科普型 | 怎么开始、怎么交付 |
| 风险科普型 | 哪些坑要避开 |

### 花絮型

| 二级类型 | 适合内容 |
|---|---|
| 团队工作花絮 | 公司真实工作 |
| 交付现场花絮 | 客户项目现场 |
| 拍摄花絮 | 内容生产过程 |
| 老板开会花絮 | 创始人判断 |
| 数据复盘花絮 | 每天怎么复盘 |

### 案例型

| 二级类型 | 适合内容 |
|---|---|
| 客户问题案例 | 客户原来卡在哪里 |
| 过程复盘案例 | 我们做了什么动作 |
| 结果展示案例 | 发生了什么变化 |
| 失败复盘案例 | 为什么没做好 |
| 行业对比案例 | 这个行业为什么这样做 |

---

## 13. 每个内容小分支必须生成的表格字段

```yaml
content_branch_fields:
  content_type: 内容大类
  sub_type: 二级类型
  target_user: 写给谁看
  pain_point: 解决什么痛点
  value: 用户看完得到什么
  title_formula: 标题公式
  cover_formula: 封面公式
  body_structure: 正文结构
  keyword: 适合关键词
  reference_accounts: 对标账号
  forbidden: 禁止事项
  cta: 行动引导
```

# 第五模块：爆文改帖

## 14. 爆文改帖的定位

爆文改帖不是照抄，而是拆解爆款笔记背后的：

```text
人群
痛点
标题结构
封面结构
正文结构
评论区问题
转化路径
```

然后换成客户自己的产品、场景和表达。

---

## 15. 爆文改帖流程

```text
找爆文
→ 拆标题公式
→ 拆封面公式
→ 拆正文结构
→ 拆评论区问题
→ 提取关键词
→ 替换客户产品
→ 替换客户人群
→ 替换客户案例
→ 合规审核
→ 发布测试
```

# 第六模块：投放体系

## 16. 投放体系定位

会议中提到蓝 V 体系除了内容，还有投放。投放不是独立动作，而是内容验证后的放大。

### 投放前提

```text
内容结构已经跑通
关键词已经明确
客户资料完整
评论区承接清楚
私域承接路径清楚
```

---

## 17. 投放表要记录

| 字段 | 说明 |
|---|---|
| 投放笔记 | 投哪条 |
| 投放目标 | 曝光 / 私信 / 表单 / 加微 |
| 关键词 | 投什么词 |
| 人群包 | 投什么人 |
| 消耗 | 花多少钱 |
| 客资 | 来多少线索 |
| 加微 | 加多少微信 |
| 成交 | 是否成交 |
| ROI | 投产比 |
| 复盘 | 继续投 / 停投 / 改标题 / 换封面 |

# 第七模块：数据复盘

## 18. 蓝 V 每日复盘

| 指标 | 说明 |
|---|---|
| 今日发布数 | 发了几篇 |
| 内容类型分布 | 知识/科普/案例/产品/营销 |
| 曝光 | 每篇曝光 |
| 点击率 | 封面标题效率 |
| 评论 | 用户问题 |
| 私信 | 咨询意向 |
| 加微 | 私域承接 |
| 成交 | 销售结果 |
| 高价值评论 | 可进入 FAQ |
| 可复制模板 | 哪条值得复制 |
| 明日优化 | 下一步做什么 |

# 第八模块：Codex调用规则

## 19. Codex生成蓝V内容时的前置判断

```yaml
bluev_content_precheck:
  client_profile: 客户是谁？
  product_or_service: 卖什么？
  benchmark: 对标账号是谁？
  faq: 百问百答是否完成？
  redline: 客户禁区是否明确？
  content_type: 知识/科普/花絮/故事/产品/案例/营销/避坑？
  sub_type: 二级内容类型是什么？
  target_user: 写给谁看？
  pain_point: 解决什么痛点？
  keyword: 埋什么关键词？
  proof: 用什么案例或依据证明？
  cta: 引导用户做什么？
```

## 20. 总结

蓝 V SOP 的核心不是“写内容”，而是：

```text
客户资料标准化
→ 对标拆解表格化
→ 百问百答资产化
→ 内容类型结构化
→ 爆文改帖公式化
→ 投放复盘数据化
→ 智能体持续投喂
```
