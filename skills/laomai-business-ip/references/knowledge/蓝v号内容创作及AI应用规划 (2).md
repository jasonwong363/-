# 蓝v号内容创作及AI应用规划｜老麦智能体投喂版

> 本文档根据会议记录整理，只提炼老麦本人观点、方法、判断和团队指令。其他说话人只作为会议背景，不纳入核心语料。

## 01. 本次会议核心结论

1. 蓝V内容不能只靠灵感，要拆成封面、标题、故事线、保障线、关键词、内容类别等多个模块。
2. 封面要聚焦痛点、对立、优势和人群；标题要由封面延展，负责把封面内容讲得更通俗易懂。
3. 一篇文案必须有故事线，故事线可以来自产品、案例、花絮、知识、人群切面或客户问题。
4. 蓝V内容可以拆成知识型、营销型、产品型、百问百答型、案例型、情绪型等不同类别。
5. AI不能一个窗口解决所有问题，要拆成多个专用智能体：封面标题、关键词、故事、审核、内容生成。
6. 智能体需要吃客户百问百答、对标爆文、行业关键词、违规词、品牌资料和案例资料。
7. 内容最终要回答：写给谁看、用户为什么看、看完得到什么、内容如何形成转化闭环。

## 02. 可沉淀能力模块

| 模块 | 用途 |
|---|---|
| 定位判断 | 这场内容里老麦如何判断业务、人群、平台、产品和转化方向。 |
| 内容公式 | 把老麦现场表达拆成可复用的标题、故事、案例、解决方案和金句。 |
| 团队SOP | 把会议中的具体要求转成团队每天可以执行的动作。 |
| 智能体规则 | 把老麦的判断、语气和知识结构沉淀成后续AI生成内容的约束。 |
| 产品转化 | 所有内容最终要服务课程、私董会、服务、直播或私域成交。 |

## 03. 方法论框架

### 蓝V内容拆解链路

```text
封面 → 标题 → 故事线 → 三段故事 → 保障线 → 关键词 → 内容类别 → 审核 → 发布
```

### 封面公式

```text
痛点 + 对立 + 优势 + 人群 + 视觉切面
```

### 标题公式

```text
承接封面内容，用更通俗的话解释用户痛点、利益点和观看理由
```

### 故事线公式

```text
一个人 / 一家公司 → 遇到什么问题 → 我们解决了什么 → 花了多久 → 得到什么结果
```

### AI分工公式

```text
封面标题AI + 关键词AI + 故事AI + 内容AI + 审核AI
```

## 04. 智能体理解重点

- 不要把这份会议当普通纪要，而要当成老麦的商业判断样本。
- 输出内容时要优先保留老麦的强判断、反问、直接表达和结果导向。
- 所有方法都要落到人群、痛点、场景、案例、产品和成交上。
- 遇到平台、账号、投流、内容、直播等问题时，要先判断它在整个商业闭环里的位置。
- 生成脚本前必须先判断：这条内容给谁看、为什么看、看完得到什么、用什么案例证明、最终承接什么产品。
